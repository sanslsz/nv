Frame Gallery component

The Frame Gallery displays a group of items that expand and collapse upon clicking. The content is contained within the section, making efficient use of the space.

Unique features:

  1. Accommodates 2 - 6 items.
  2. The Animated transitions bridge the gap between the collapsed and expanded states. The continuous transformation allows the audience to see the element appear, resize and reposition, providing a fresh and interactive experience.
  3. Better visual aesthetics and legibility thanks to the color theme options.

Overview

Anatomy

Image fitting options

Auto-fit

Both the collapsed and expanded states share the same setting: background-size: cover background-position: center. As a result, a smiliar crop is maintained between the 2 states.

Auto-fit is recommended for images with evenly distributed content, such as pattern, abstract art, certain concept artworks and photographs.

Custom crop

The collapsed and expanded state have very different background size and position settings, resulting in a zoom-in or zoom-out effect between the 2 states.

It’s a good choice for images with a generous amount of empty space around the subject, such as product renderings. The subject is usually positioned in an asymmetrical position.

Item number and layout

The component supports 2-6 items and all 4 viewports. The number of items and viewport size determine the layout.

Specification

Color Theme

There are 3 color theme options:

  1. Dark(default): recommended if the image has a dark color palette.
  2. Light: recommended if the image has a light color palette.
  3. Custom: a natural blend with colors sampled from the image.

Color contrast enhancement

An optional text legibility enhancement feature can be enabled to help boost the color contrast between the text and background image. When enabled, in the expanded state on phone viewport, the item's background gradient height is increased to 300%, expanding the coverage of the dark end of the gradient. If the item meets the following description, it may be a good candidate for this feature:

  1. Large body of text that fills up the entire item in the expanded state on phone viewport.
  2. The background image has an opposite or mixed color palette that doesn’t provide enough contrast against the text.

Interaction

Hover

The cursor changes from default to pointer. The image scales up 5%.

Click

Initiates an animated transition between the collapsed and expanded states. The background image resizes and repositions, content appears and disappears.

Collapsed State

The heading and subhead are displayed. All items are in a collapsed state when the page is first loaded.

Expanded State

The description, CTAs and close buttons are revealed below the heading and subhead. In the desktop, laptop and tablet viewports, an expanded item takes up the entire section space. In the phone viewport, an expanded item’s height increases, forming an aspect ratio of 4:6.

Animation Sequence - Expand

Timeline Timing Function Element Effect
0-0.2s Ease-out Other / non-active items Fade-out
0.2-0.5s Ease-out Active item Expand to cover full section
Ease-out Active item’s image Resize and reposition
0.5-0.8s Ease-out Description text, CTA, close button Fade-in

Animation Sequence - Collapse

Timeline Timing Function Element Effect
0-0.3s Ease-out Description text, CTA, close button Fade-out
0.3-0.6s Ease-out Active item Collapse to original item size
Ease-out Active item’s image Resize and reposition
0.6-0.8s Ease-out Other / non-active items Fade-in

Overflow scroll:

In the expanded state, when there’s not enough vertical space to accommodate the content, the description text becomes scrollable. A scrollbar appears to indicate text overflow. The top and bottom fading effects help the text blend in, and suggest scrolling directions. Heading, subhead and CTAs do not scroll.

Guidelines

Accessibility

Alternate text for images

Designers usually don’t deal with alternate text for images, which describes the image and helps SEO and assisted devices understand the image content. This option is available in the backend configuration of the component.

Color theme and text legibility

Leverage the image’s color palette and choose a color theme that maximizes the color contrast between the text and background image. Use the “legibility boost” feature for phone viewport if necessary.

Image naming convention

Example image file name: [rename-me]-fg-i1of6-p.jpg
-fg: frame gallery
-cc: custom crop
-i1of6: item number 1 of the 6 items
-p: phone
