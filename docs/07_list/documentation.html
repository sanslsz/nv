List Component

There are a variety of list types within the list family. In general, a list component itemizes things under the same category, transforming the content into digestible bits of information. The list components help the audience read and process the information efficiently by establishing scannable structure and hierarchy within the page. View Prototype

Overview

Bullet List

A bullet or unordered list houses items that are of similar importance or under the same category. The items don’t have a numerical sequence, meaning changing their order doesn’t affect the accuracy of the information.

Default

We do not use a bullet glyph, instead a “>” (greater than) character is placed on the left side of the list items.

Custom Icon

Custom icons replace the greater than characters.

Numbered List

A numbered or ordered list houses items that are of similar importance or under the same category. The sequence or priority among the list items is conveyed by the lead character - a number and period appear on the left side of each item.

Default

The default numbered list is the most commonly used.

Titled

A subhead gives an entry point to each list item. It is a good option for items that need additional description.

Custom Counter

Custom counter list allows the designer to define the lead character. Commonly used in content with multiple level list structure such as user agreements, legal terms and table of contents etc. In the following example, there are 3 levels of items. For List items at level 2 and 3, the lead numbers include the upper level’s numbers.

Multi-level / Nested List

The default bullet and numbered lists can be nested inside each other, creating a multi-level list structure. A maximum level of 3 is recommended.

Description List

Description term and detail groups are housed in the description list. A description term is followed by a description detail that provides explanation.

Specification

Content in the specification list component features a group of property and value pairs that provide specification related information for the products. An optional group title can be added to the specification list.

Glossary

Content in the glossary list component features terminology and definition pairs that are organized in alphabetical order.

Event List

The event list component features information for a group of events. Each list item compromises the date, title, venue and detail of an event. The event list items are arranged in chronological order.

Guidelines

Headings and List Components

Heading and list are important mechanisms that often work together to help establish the hierarchy within the page and make content scannable. Consider the overall structure of the page and avoid repeated content that may cause confusion.

When to Use

Not all content can benefit from the list structure. Consider the following questions before turning something into a list, or when receiving a list from the copywriter:

  1. Does breaking the content down to list items help the audience read and understand the information?
  2. Do the list items share a common topic or category?
  3. Does sequence exist among the list items?
  4. Is there a pattern of pairing such as property and value, term and definition, time and location?

For comparison among the items consider using table for its 2 directional display capability.

Content Flow

List items by default are block level elements, taking the full width and stacking vertically. For lists without nested items, we can change its content flow and enhance the design with layout options.
