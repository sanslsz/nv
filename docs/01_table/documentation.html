Size

Tables have an interesting mechanism that manages the width. By UA default, a table is only as wide as its content, up to the full container width. Each column takes up the space that’s necessary to accommodate the content within it. However our tables are set to be full container width on large viewports, having each column divide the available space within. See below for size and space distribution options.

UA default Desktop Laptop Tablet Phone
Table width Content width 100% container width 100% container width 100% container width, or min-width / fixed width(may overflow), or content width(may overflow)
Column width Content width Auto(flexible), fixed size in px, percentage
Table length Content length Content length Content length Max-height: 90vh (may overflow)

Small viewport

When the table’s width or height is bigger than its container, overflow occurs on their respective axis. The treatments below are available to optimize the responsiveness under small viewports.

Horizontal scrolling

Allows the users to scroll the overflowed content both horizontally and vertically. It’s recommended that the table’s width is below 800px to avoid excessive scrolling.

  1. Sticky row header Sticks to the left with a drop shadow on the right side. The elevation gives it a sense of depth and suggests that the rest of the table can be tucked under.
  2. Overflow on the right The absence of the right padding suggests overflow to the right. Once it’s scrolled to the end the padding is restored.
  3. Scroll bar Indicates the position of the visible content in relation to the full content.

Convert to list

The tabular structure gets rearranged into a vertical list structure. TTHis option is recommended for tables with more than 50 cells.

Hide columns

Hide the row headers and reposition them as rows to reduce the width of the table.

In addition to the column reposition described above, this video shows that horizontal scrolling doesn’t start until the viewport width is 515px or smaller. Because the left 2 columns are removed, the space is enough for the remaining columns(480px min-width) when the viewport is larger than 515px. It’s not necessary to enable horizontal scrolling until the table overflows the container.

Interaction

Interactive features give the audience control on how the table content is presented, making it easy to narrow down and pinpoint relevant information.

Search

Identifies matches from the filter criteria, displays them in the dropdown along with the filter category in bold weight. Selecting the item applies the filter.

Filters

Dropdown selects that target a specific table column. It can be placed above the table or incorporated into the table head.

Chips / pills

Display the filter criteria applied to the table. Users have the option to remove a filter criteria by clicking the "X", or clear all filter criteria.

Pagination

Set the number of the items to display per page, and page navigation.

Sticky table head

Header row sticks to the top of the viewport while scrolling, keeping the context in place.

Sticky cell content

Content within the cell sticks to the top of the viewport, below the table head. This helps relevant information within the same row stay visible and/or close to each other while scrolling.

The 1st, 2nd, 4th and 5th columns all have their content set to sticky. They stay visible while the 3rd column scrolls, keeping the context in close proximity.

Tips

  1. Two-directional data display and comparison is the table's strength. For simple property and value pairs, consider the spec list component.
  2. For e-commerce pages, it may be better to use card items and a side filter bar, instead of a table with filter functionality.
  3. Depending on the content, some tables benefit from being simplified and restructured. Under certain viewports, the columns and rows can be removed and/or repositioned. Elements inside the table cells can be sticky to keep relevant information visible while scrolling.

Backend configuration

The structure and layout of the tables depends on their content. The backend features are designed for the table component on Adobe Experience Manager CMS. Supporting content simple and complex, the configuration settings allow for maximum customization while maintaining the component's function, visual and interaction.
